====================================
ROOT CAUSE ANALYSIS (LANJUTAN)
====================================
Penyebab berikutnya kegagalan deteksi:
1. Ketiadaan Rules Kustom: Meskipun antarmuka enp0s8 sekarang dipantau dengan benar, Nmap tidak memicu peringatan karena file 'custom.rules' (yang seharusnya berisi aturan spesifik deteksi scan) tidak ada di /etc/suricata/rules/ dan tidak dideklarasikan di dalam suricata.yaml.
2. Default Rules Tidak Memadai: Rules Emerging Threats default (suricata.rules) gagal mendeteksi Nmap SYN scan dasar tanpa konfigurasi agresif, sehingga rules lokal absolut diperlukan.

Langkah Perbaikan Berikutnya yang Dibutuhkan:
- Membuat file /etc/suricata/rules/custom.rules di VM.
- Mengisi file tersebut dengan SID khusus untuk mendeteksi Nmap.
- Mendaftarkan ' - custom.rules' di dalam suricata.yaml pada bagian rule-files.
- Restart Suricata.
